# lumx-site-demo — Package Knowledge Base

Private Gatsby + MDX documentation site. Published at design.lumapps.com. **Not an npm package.**

## STRUCTURE

```
packages/site-demo/
├── content/product/     # MDX documentation source (the actual docs)
│   ├── components/      # Per-component MDX pages
│   ├── foundations/     # Color, typography, spacing, iconography, theme
│   ├── patterns/        # UI patterns (form validation, drag-drop, loading…)
│   ├── utilities/       # CSS utility class docs
│   └── design-tokens/   # Token reference pages
├── src/
│   ├── components/      # Gatsby/React site components (PropTable, DemoBlock…)
│   ├── templates/       # Gatsby page templates
│   └── utils/           # Site utilities
├── scripts/
│   └── generate-demo-stories.mjs  # Generates Storybook stories from MDX
└── static/demo-assets/  # Static images and assets for demos
```

## WHERE TO LOOK

| Task                           | Location                                             |
| ------------------------------ | ---------------------------------------------------- |
| Add/edit component docs        | `content/product/components/<name>/`                 |
| Add framework demo (React/Vue) | `content/product/components/<name>/react/` or `vue/` |
| Edit PropTable data            | MDX frontmatter or component JSDoc                   |
| Add pattern page               | `content/product/patterns/`                          |
| Edit site navigation           | `content/menu.yml`                                   |

## COMMANDS

```bash
yarn start          # Dev server on :4000 (from repo root)
yarn build:site     # Production Gatsby build
```

## GENERATED FILES

-   `src/stories/demos-generated/` in lumx-react and lumx-vue — generated by `scripts/generate-demo-stories.mjs`
-   `public/` — Gatsby build output — never commit
-   Run `yarn workspace lumx-site-demo generate:demo-stories` after editing MDX demos

## NOTES

-   Gatsby uses MDX v2; standard `.mdx` extension
-   Sandpack is used for interactive code sandboxes in some pages
-   `frontVersion` URL param can override version shown in docs
-   `static/demo-assets/` images are NOT generated — commit them
-   The site consumes published `@lumx/react` and `@lumx/vue` (peer deps), not local source
