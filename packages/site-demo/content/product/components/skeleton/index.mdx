# Skeleton

**Skeleton are primitives shapes which mimic a piece of content in a recognizable way. It communicates [loading states](/product/patterns/loading-state).**

There are three variants: `circle`, `rectangular`, `typography`. When designing loading states with skeletons, use a combination of variants that best mimic the replaced content.

## Circle

Use circle skeletons to mimic components like [avatar](/product/components/avatar/), [icon](/product/components/icon). Respect at best the size of replaced element using the T-shirt [sizes](/product/foundations/size).

<DemoBlock orientation="horizontal" withThemeSwitcher gap="huge" demo={{ react: Demo1 }} />

## Rectangle

Use rectangular skeletons to mimic [thumbnail](/product/components/thumbnail), [button](/product/components/button), [chip](/product/components/chip), [text field](/product/components/text-field) and so on. `width`, `height` and `border-radius` are adjustable.

<DemoBlock orientation="horizontal" withThemeSwitcher gap="huge" demo={{ react: Demo2 }} />

## Typography

Use typography skeletons to mimic text. Use [text style](/product/foundations/typography) as parameter to easily set a replacement for texts. `width`, is adjustable.

<DemoBlock withThemeSwitcher demo={{ react: Demo3 }} />

## Animation and style

Skeletons have a unique animation and style: pulsating grey or white depending on the theme.

<DemoBlock orientation="horizontal" withThemeSwitcher demo={{ react: Demo4 }} />

### Accessibility concerns

Skeletons are purely visual and do not use any special accessibility or landmark role.

Please look into ARIA live regions to announce the loading state and changes to the user.
