# List

**Lists group items vertically.**

## Item layout

List items has three sections.\
The main section that can display simple text or multi-line text,
the `before` section usually used to display an icon or thumbnail and the `after` section usually used to
display icons or actions.

## Item sizes

List items come in 4 sizes: `tiny`, `regular`, `big` and `huge`.

`tiny`, `regular` and `big` list items have specific **minimum heights** and a `8px` vertical padding.\
Favor those 3 sizes for lists with list items with consistent heights.

`huge` list items have a `16px` vertical padding.\
Use this size for any other cases.

The list horizontal padding has 2 sizes: `big` and `huge`.

## Clickable

List items can be made clickable either as a link or a button.\
Clickable list items can then be disabled, highlighted or selected.
Use clickable list items when users need to select one or more options in a list.

## List sections

Use sections to group list items into labeled sections. Each section can have a `label` and an optional `icon`.
Adjacent sections are automatically separated by a divider.

### Accessibility concerns

The `List` component renders a standard `<ul>` element with an implicit `list` role, and `ListItem`
renders a `<li>` element with an implicit `listitem` role. Screen readers will announce the list
and its items natively.

#### Clickable list items

When a `ListItem` is made clickable (via `onClick`, `linkProps.href`, or `linkAs`), its
inner content is wrapped in a button or a link depending on the props provided. This ensures
clickable items are keyboard accessible and properly announced by screen readers as interactive elements.

Clickable list items can be disabled in two ways:

-   `isDisabled` which completely disables the item. It's not focusable and pointer events (hover, click,
    etc.) are all disabled too. You'll have to find another way to indicate to users that the item is here and why it
    is disabled.
-   `aria-disabled` only disable changes on the item (read only) so it's still focusable and accessible. You'll have
    to setup either a tooltip or an aria description (must be visible too) to explain to the user why it is disabled.

#### Clickable list items with nested interactive elements

When a list item needs **multiple independently clickable elements**, use `ListItem.Action` instead.
It activates the [action area](../../utilities/action-area) pattern: the primary action expands its click target to
the full list item area, while other interactive elements in `before` and `after` slots remain separately focusable
and clickable. This keeps the DOM valid and preserves a clear tab order for keyboard and screen reader users.

#### Sections

`ListSection` with a `label` renders a standard HTML nested list: a `<li>` containing a labeled
`<ul>` (via `aria-labelledby`). This is the recommended way to split a list into named sections.

Pass `itemsWrapperProps={{ role: 'group' }}` to change the inner `<ul>` to a `group` role.
Use this when sections represent grouped items/options inside a `listbox` or `menu`.

**Patterns to avoid:**

-   **`ListSection` without a `label`**: produces an unnamed nested list with a "phantom" list item
    wrapper, which is semantically confusing for screen readers.
-   **`ListSubheader`** (deprecated): renders a `<li>` used as a section header, producing an improper
    list structure. Use `ListSection` with a `label` instead.

#### Divider

`ListDivider` and the automatic dividers between `ListSection` are purely visual. Only a section with label provides
a correct split of items in a list.
