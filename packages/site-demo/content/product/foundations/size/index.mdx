# Size

**Sizes are names given to steps of dimension scales. They help design with visual consistency and give imaged names to easily refer to.**

There are two scales, **T-shirt** scale, and **Abstract** scale.

## T-shirt

T-shirt sizes correspond to standardised values across all the design system. T-shirt sizes go from `xxs` to `xxl`. Use t-shirt sizes to define precise and invariable dimensions to elements. An element doesn’t have to exist in each of every size.

## Abstract

Abstract sizes (`tiny`, `regular`, `medium`, `big`, `huge`) don’t correspond to standardised values across the design system, they represent size variations at a component level. They can be values like 400px or a specific behavior like full-screen with margins (see [Dialog](/product/components/dialog)). Use Abstract sizes to define elements dimensions that don’t need to or can’t be standardised. A component doesn’t have to exist in each of every size.
